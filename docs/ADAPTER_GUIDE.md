# 适配器开发指南

更新时间：2026-04-03

## 1. 设计目标

适配器用于把“站点行为”映射为“统一 API 能力”，确保上层调用协议稳定，下层可替换。

## 2. 能力抽象建议

每个适配器建议声明以下内容：

- 支持模型列表
- 支持输入类型（text / image / video）
- 图片策略（required / optional / forbidden）
- 返回类型（text / image / mixed）

## 3. 开发约定

- 输入统一走 OpenAI 风格 `messages`
- 错误统一包装为标准错误对象
- 不把站点特定字段直接泄漏给上层
- 必要时提供 `keepAlive`，降低会话过期概率

## 4. 并发与稳定性

- 支持配置最小调用间隔与抖动
- 失败需区分“可重试”与“不可重试”
- 与任务队列/故障转移策略配合

## 5. 测试清单

- 基础连通：模型可见、单次调用成功
- 多模态：图文输入与边界数量校验
- 异常路径：登录过期、页面超时、风控拦截
- 长稳运行：队列积压、重试、恢复能力

## 6. 发布建议

- 先灰度给单实例
- 观察错误码分布和耗时
- 确认无回归后再扩大流量
